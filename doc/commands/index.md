# 索引文档到向量数据库命令

将思源笔记文档索引到向量数据库，支持自动分块处理。

## 命令格式

```bash
siyuan index [options]
```

## 参数说明

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `--notebook <id>` | string | ❌ | 指定笔记本 ID |
| `--doc-ids <ids>` | string | ❌ | 指定文档 ID 列表（逗号分隔） |
| `--force` | boolean | ❌ | 强制重建索引 |

## 使用示例

### 索引所有笔记本
```bash
# 索引所有笔记本的文档
siyuan index
```

### 索引指定笔记本
```bash
# 索引指定笔记本
siyuan index --notebook 20260227231831-yq1lxq2
```

### 索引指定文档
```bash
# 索引指定文档
siyuan index --doc-ids "doc-id-1,doc-id-2"
```

### 强制重建索引
```bash
# 强制重建索引
siyuan index --force
```

## 向量索引特性

### 自动分块处理
当文档内容超过 4000 字符时，系统会自动使用思源笔记 API 的块列表功能将文档分块索引。

### 智能分块策略
基于文档的块结构（标题、段落、列表等）进行分块，每个块最大 3000 字符。

### 保留原始信息
分块索引时会保留原始文档 ID，搜索时可以追溯到原始文档。

### 递归处理
支持递归处理子文档，确保所有内容都被正确索引。

## 返回格式

```json
{
  "success": true,
  "data": {
    "indexed": 150,
    "skipped": 20,
    "errors": 0,
    "total": 170
  },
  "message": "索引完成",
  "timestamp": 1646389200000
}
```

## 注意事项

1. **向量搜索配置**：需要配置 Qdrant 和 Embedding 服务
2. **自动分块**：超长文档会自动分块，无需手动处理
3. **索引时间**：首次索引可能需要较长时间，取决于文档数量
4. **增量索引**：默认只索引新增和修改的文档，使用 `--force` 可强制重建
5. **权限限制**：需要相应的权限才能索引文档
